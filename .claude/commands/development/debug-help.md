# /debug-help

デバッグ支援を行い、問題の特定と解決策を提供します。

## 実行内容
1. エラーメッセージの解析
2. ログの分析と問題箇所の特定
3. 再現手順の確認
4. 原因の推定と修正案の提示
5. テストケースの作成支援
6. デバッグ手法の提案

## 使用例
- `/debug-help` - 現在のエラー状況を分析
- `/debug-help "error message"` - 特定のエラーメッセージを分析
- `/debug-help --logs` - ログファイルから問題を特定

## 対応する問題タイプ
- コンパイル/ビルドエラー
- 実行時エラー・例外
- パフォーマンス問題
- メモリリーク
- データ破損・不整合
- ネットワーク・通信エラー
