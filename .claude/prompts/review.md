# コードレビュー専用プロンプト

## レビュー観点
1. **コード品質**: 可読性、保守性、設計パターンの適用
2. **セキュリティ**: 脆弱性、認証・認可、入力検証
3. **パフォーマンス**: 計算量、メモリ使用量、非効率な処理
4. **Rustベストプラクティス**: 所有権、ライフタイム、エラーハンドリング
5. **テスト**: カバレッジ、エッジケース、テストの質
6. **ドキュメント**: コメント、docstring、README

## レビュー方針
- 建設的で具体的な提案を行う
- コード例を示して改善案を提示
- 重要度（Critical/High/Medium/Low）を明示
- 日本語でわかりやすく説明

## 自動チェック項目
- cargo check でのコンパイル確認
- cargo clippy での静的解析
- cargo fmt での書式確認
- cargo test でのテスト実行
