# Claude Code Review Action

このドキュメントでは、専用のClaude Code Review Actionの使用方法について説明します。

## 概要

`claude-review.yml` ワークフローは、プルリクエストのコードレビューに特化したClaude Code Actionです。通常の `claude.yml` とは独立して動作し、レビューに最適化された設定とプロンプトを使用します。

## 主な機能

### 自動トリガー
- **Pull Request作成時**: 新しいPRが作成されると自動でレビューを実行
- **Pull Request更新時**: PRに新しいコミットがプッシュされると自動でレビュー
- **@claude コメント**: PR内で `@claude` とコメントすると手動でレビューを実行

### レビュー観点
1. **コード品質**: 可読性、保守性、設計パターンの適用
2. **セキュリティ**: 脆弱性、認証・認可、入力検証
3. **パフォーマンス**: 計算量、メモリ使用量、非効率な処理
4. **Rustベストプラクティス**: 所有権、ライフタイム、エラーハンドリング
5. **テスト**: カバレッジ、エッジケース、テストの質
6. **ドキュメント**: コメント、docstring、README

### 専用設定ファイル
- `.claude/settings-review.json`: レビュー専用の設定
- `.mcp-review.json`: レビュー用MCP設定
- `.claude/prompts/review.md`: レビュー用プロンプトテンプレート

## 使用方法

### 1. 自動レビュー
プルリクエストを作成または更新すると、自動的にレビューが開始されます。

### 2. 手動レビュー
プルリクエストのコメント欄で以下のようにコメントします：

```
@claude このPRをレビューしてください
```

```
@claude セキュリティ観点でレビューを実行
```

```
@claude パフォーマンスの問題をチェック
```

### 3. 特定ファイルのレビュー
```
@claude src/main.rs のレビューをお願いします
```

## レビュー結果の形式

レビューコメントは以下の形式で提供されます：

### 👍 良い点
- 具体的な良い実装や設計について

### ⚠️ 改善提案
- **[重要度]** 問題の説明
  ```rust
  // 改善前
  [現在のコード]
  
  // 改善後
  [提案するコード]
  ```
  理由: 改善理由の説明

### 🔍 確認事項
- [ ] 確認が必要な項目

### 📝 その他のコメント
- その他の気づきや提案

## カスタマイズ

### レビュープロンプトの変更
`.claude/prompts/review.md` を編集してレビューの観点や方針を変更できます。

### MCP設定の調整
`.mcp-review.json` でレビュー時に使用するMCPサーバーを調整できます。

### 設定の変更
`.claude/settings-review.json` でレビューモードの設定を変更できます。

## 通常のClaude Actionとの違い

| 項目 | claude.yml | claude-review.yml |
|------|------------|-------------------|
| 目的 | 一般的な開発支援 | コードレビュー特化 |
| トリガー | @claude コメント | PR作成/更新 + @claude |
| 設定ファイル | .claude/settings.json | .claude/settings-review.json |
| プロンプト | 汎用プロンプト | レビュー専用プロンプト |
| 権限 | write権限あり | 主にread権限 |

## トラブルシューティング

### レビューが実行されない
1. ワークフローファイルが正しく配置されているか確認
2. 必要なシークレット（ANTHROPIC_API_KEY等）が設定されているか確認
3. GitHubActionsの実行権限があるか確認

### レビュー品質を向上させたい
1. `.claude/prompts/review.md` でレビュー観点を詳細化
2. `.claude/settings-review.json` でレビューモードの設定を調整
3. プロジェクト固有のレビューチェックリストを追加
